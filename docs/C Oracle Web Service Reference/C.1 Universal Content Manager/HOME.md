# C.1 Universal Content Manager


### Overview

UCM is a document repository used by Oracle as an interface between the ERP system and boundary applications.  All Jobs in oracle read from and write to UCM to prevent dependencies on remote systems.  They provide a SOAP API which allows interaction with this repository.

#### Connection Basics

* Use of integration account `svc.INTEGRATE`.  Send via Basic Authentication.
* Base Path within Oracle Instance: `/idcws/GenericSoapPort`
* WSDL Path: `/idcws/GenericSoapPort?WSDL`
* SoapAction: `urn:GenericSoap/GenericSoapOperation`

### SOAP Interaction Basics

Oracle Documentation: <https://docs.oracle.com/cd/E23943_01/doc.1111/e11011/c04_core.htm#CSSRG2076>

* Generic service endpoint
* IdcService indicates operations
* All requests and responses use generic key-value pair results...no operation-specific XML schema.
* All operations return a `multipart/related` response type.  XML (and other) response components must be extracted before they can be used.  `(?s).*(<env:Envelope .*</env:Envelope>).*` can be used to exclude an XML response.
  * If multiple parts, the MIME boundary must be extracted from the Content-Type HTTP header and used to split the response.

#### Common Document Properties

| Property Name     | Meaning                                                                                |
| ----------------- | -------------------------------------------------------------------------------------- |
| `dID`             | Unique, integer, assigned value internal to UCM.  Differs between file versions.       |
| `dDocName`        | Unique assigned name within the UCM file system. Same Across file versions             |
| `dDocTitle`       | Visible title of the file.  This appears in the UCM UI and is used for all file scans. |
| `dFormat`         | The oracle-assigned mime type of the file.                                             |
| `dDocCreatedDate` | `M/d/YY h:mm a` formatted date.                                                        |


#### Common Operations

| Operation Name       | Purpose                                                                       |
| -------------------- | ----------------------------------------------------------------------------- |
| `GET_FILE`           | Downloads a single file by document ID from UCM.                              |
| `UPDATE_DOC_INFO`    | Updates metadata on a file, used for renaming.                                |
| `GET_SEARCH_RESULTS` | Searches for files matching given criteria, usually by document title prefix. |
| `PING_SERVER`        | Checks if server is up and credentials are valid                              |

### SOAP Operation Details

#### `GET_FILE`

* **Required Inputs**
  * `dID` : Numeric identifier of a file.
* **Output**
  * `multipart/related` Output
    * Part 1: XML response
    * Part 2: Requested File

##### Sample SOAP Envelope

```xml
<?xml version="1.0" encoding="utf-8"?>
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Body>
    <GenericRequest xmlns="http://www.oracle.com/UCM" webKey="cs">
      <Service IdcService="GET_FILE">
        <Document>
            <Field name="dID">1234567</Field>
        </Document>
      </Service>
    </GenericRequest>
  </soap:Body>
</soap:Envelope>
```

##### Sample Response

> File data is in the 2nd part of the multipart message.  Multipart boundary is in the `Content-Type` HTTP Header.
> Extract from first group with regex: `^.*boundary="([^"]+)".*$` (set as variable `mime.boundary` in the example below)
> Might be able to then extract file contents using group 2 of Regex:
> `(?s)--${mime.boundary}.+?\r\n\r\n(.+?)--${mime.boundary}.+?\r\n\r\n(.*?)--${mime.boundary}--`
